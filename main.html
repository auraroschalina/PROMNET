Version Control System Dengan Git

Version Control System (VCS) adalah sebuah sistem yang melakukan source code management (SCM) untuk mengelola perubahan di setiap dokumen, program komputer, website, dan kumpulan pemrograman lainnya.

Pengertian Git

Git adalah sebuah software berbasis Version Control System yang digunakan untuk mencatat, mengelola perubahan pada seluruh file atau repository sebuah proyek. Tujuannya agar semua orang yang terlibat dalam penyusunan kode bisa menyimpan database tersebut juga membantu antar divisi proyek untuk memantau dan menghubungkan antar ekstensi yang berbeda dengan mudah sehingga aplikasi yang dibuat oleh tim proyek dapat berfungsi tanpa harus menghubungkannya secara manual. Sedangkan Github adalah sebuah layanan cloud (repository) yang berguna untuk menyimpan dan mengelola sebuah proyek.

Perbedaan Git dan Github

Perbedaan yang utama antara Git vs Github adalah pada bentuk pengoperasian dan fungsinya. Keduanya sama-sama memudahkan penggabungan dan pembagian kode, namun bedanya Git merupakan open source software yang dirancang untuk merangkum riwayat sumber dari coding. Sedangkan Github merupakan layanan hosting berbasis web sebagai repositori Git. Github juga menawarkan seluruh DVCS SCM yang diperkaya dengan berbagai fitur seperti manajemen tiket, manajemen proyek, hingga pelacakan bug. Melalui Github developer bias membagikan, mengakses, dan menyimpan salinan repositori dari jarak jauh.

Comand Dasar pada Git

  • git config --global user.name "[nama]" = Mengatur nama yang ingin ditautkan pada transaksi commit
  • git config --global user.email "[alamat email]" = Mengatur alamat surel yang ingin ditautkan pada transaksi commit
  • git init [nama-proyek] = Membuat repositori lokal baru dengan nama tertentu
  • git status = Daftar semua berkas baru atau modifikasi yang siap didaftarkan dalam commit
  • git add [berkas] = Rekam berkas yang akan didaftarkan ke dalam commit
  • git commit -m "[pesan]" = Daftarkan perubahan berkas secara permanen di riwayat revisi
  • git remote add origin <link repo> = Menambahkan remote repository pada local folder.
  • git push origin <branch> = Unggah semua commit dari cabang lokal ke GitHub.
  • git branch [nama-cabang] = Membuat cabang baru
  • git branch -a = Melihat daftar branch yang sudah ada pada repository
  • git checkout [nama-cabang] = Berpindah ke cabang tertentu dan perbarui direktori yang sedang dikerjakan
  • git pull origin <branch> = Mendapat update dari repository yang dikaitkan dengan remote.
  • git log = Daftar riwayat revisi untuk cabang saat ini
  • git fetch [remote] = Unduh semua riwayat dari repositori remote
  • git clone <link repo> = Melakukan cloning dari repo git ke repo local
